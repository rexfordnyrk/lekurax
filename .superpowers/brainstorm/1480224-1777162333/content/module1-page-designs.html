Module 1 — Page designs (layout + workflow)

This screen locks the UI contract for the four Admin pages. It’s based on your mockup’s structure (sidebar + top bar + data-dense content), and your decision to make each item a separate page under Admin.

Global layout contract (all Admin pages)
Common chrome
  • App sidebar: Admin section contains links to the 4 pages.
  • Top bar: Breadcrumb “Admin → {Page}”, page title, right-side primary action (page-specific).
  • Content padding: 24px desktop; responsive down to 375px with tables becoming horizontal scroll or stacked rows.
  • Data-dense style: KPI cards + filter bar + table card (hover row highlight).
  • Icons: use Remixicon/Lucide-style (no emojis).
  • Theme: light-first with pharmacy-green primary; dark mode optional later (not required for Module 1).
1) Admin → Users
/admin/users
KPI row (Total users • Active • MFA enabled • Active sessions)
Search (name/email/phone)
Role filter
Status filter
More filters
Users table: User • Role • Branch • Status • Last login • MFA • Actions
Primary action: “Add user” → modal.
Modals: Create user, Edit user, View user (with suspend/deactivate).

Must support

  • Create/edit/suspend user
  • Branch assignment (single or multi; matches Authz capabilities)
  • MFA required flag at user level (if supported)

Nice-to-have later

  • Export users CSV
  • Bulk actions
  • Advanced column customization
2) Admin → Roles & Permissions
/admin/roles
Table card: Role name • Description • Type (System/Predefined/Custom) • Users • Permissions summary • Actions
Primary action: “Create custom role” → modal.
Modal: role details + permission matrix grouped by module (checkbox grid).
Permission matrix: grouped sections + checkboxes + save/cancel

Must support

  • View system/predefined roles (read-only)
  • Create/edit custom roles
  • Permission grouping by modules/features

Nice-to-have later

  • Role versioning / change history
  • “Compare roles” view
3) Admin → Audit logs
/admin/audit
Search audit logs
Category filter
Time range
Export
Table: Timestamp • User • Action • Category • IP • Details
Primary action: “Export logs” (CSV/JSON).
Details: row “view” opens side panel or modal with full payload.
4) Admin → Auth policies (tenant security settings)
/admin/auth-policies
Single page, 3 cards, one save flow
Password policy card (min length, complexity, expiry, history, lockout)
MFA policy card (required/optional, allowed methods, grace period)
Session policy card (idle timeout, max session age, device limit)
Primary action: “Save policies” (with validation + success toast).
